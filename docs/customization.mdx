fromsrc is designed to be customized. you own the components.

## component overrides

wrap any component to add custom behavior:

```tsx
import { Callout as CalloutBase } from "fromsrc/client"

export function Callout(props) {
  return <CalloutBase className="my-custom-class" {...props} />
}
```

## mdx components

pass custom components to the content renderer. first define your components:

```tsx
const components = {
  h1: (props) => <h1 className="custom-h1" {...props} />,
  a: (props) => <a className="custom-link" {...props} />,
}
```

then use them with the content component in your page:

```tsx
<Content source={content} components={components} />
```

## sidebar navigation

build your own navigation structure:

```tsx
const navigation = [
  {
    title: "guides",
    items: [
      { type: "item", title: "intro", href: "/docs" },
      {
        type: "folder",
        title: "advanced",
        items: [
          { type: "item", title: "plugins", href: "/docs/plugins" },
        ],
      },
    ],
  },
]
```

## search

customize search behavior by wrapping the component:

```tsx
export function CustomSearch({ docs }) {
  return <Search docs={docs} basePath="/documentation" />
}
```

## full control

since fromsrc exports primitives, you can:

- use only the components you need
- replace any component entirely
- add your own components
- modify styles via css variables
- build custom layouts

no framework lock-in. no hidden abstractions.
