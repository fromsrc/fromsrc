Lightweight inline notes for quick annotations.

## Note

Simple aside for additional context:

```mdx
<Note>this feature requires version 2.0 or later</Note>
```

<Note>this feature requires version 2.0 or later</Note>

## Important

Highlight important information:

```mdx
<Important>make sure to backup your data before proceeding</Important>
```

<Important>make sure to backup your data before proceeding</Important>

## Experimental

Mark experimental features:

```mdx
<Experimental>this API may change in future versions</Experimental>
```

<Experimental>this API may change in future versions</Experimental>

## Vs callout

Use `Note` for brief inline annotations. Use `Callout` for longer, more prominent messages that need icons and titles.
