Display version history with categorized changes.

## Usage

```mdx
<Changelog>
  <Release version="2.0.0" date="2026-01-15">
    <Change type="added">dark mode support</Change>
    <Change type="changed">improved search performance</Change>
    <Change type="fixed">sidebar collapse animation</Change>
  </Release>
  <Release version="1.5.0" date="2026-01-01">
    <Change type="added">keyboard navigation</Change>
    <Change type="deprecated">old search API</Change>
  </Release>
</Changelog>
```

<Changelog>
  <Release version="2.0.0" date="2026-01-15">
    <Change type="added">dark mode support</Change>
    <Change type="changed">improved search performance</Change>
    <Change type="fixed">sidebar collapse animation</Change>
  </Release>
  <Release version="1.5.0" date="2026-01-01">
    <Change type="added">keyboard navigation</Change>
    <Change type="deprecated">old search API</Change>
  </Release>
</Changelog>

## Change types

| type       | color  | use for          |
| ---------- | ------ | ---------------- |
| added      | green  | New features     |
| changed    | blue   | Modifications    |
| fixed      | amber  | Bug fixes        |
| removed    | red    | Removed features |
| deprecated | orange | Deprecated APIs  |
| security   | purple | Security fixes   |

## All types

<Changelog>
  <Release version="1.0.0">
    <Change type="added">new feature</Change>
    <Change type="changed">updated behavior</Change>
    <Change type="fixed">bug fix</Change>
    <Change type="removed">removed feature</Change>
    <Change type="deprecated">deprecated API</Change>
    <Change type="security">security patch</Change>
  </Release>
</Changelog>
