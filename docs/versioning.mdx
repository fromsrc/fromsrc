Document multiple versions of your project.

## Strategies

### Branch-based (recommended)

Separate git branches per major version:

```
main        → docs.site.com (latest)
v2          → v2.docs.site.com
v1          → v1.docs.site.com
```

Benefits:

- No content duplication
- Independent builds per version
- Clean git history

### Directory-based

Version folders in single branch:

```
docs/
  v3/          → current
  v2/          → previous
  v1/          → legacy
```

Benefits:

- Single deployment
- Easy diffing between versions

## Version selector

Add a version dropdown to your docs:

```tsx
import { VersionSelect } from "fromsrc/client";

<VersionSelect
  current="v3"
  versions={[
    { id: "v3", label: "v3 (latest)", href: "/" },
    { id: "v2", label: "v2", href: "https://v2.docs.site.com" },
    { id: "v1", label: "v1", href: "https://v1.docs.site.com" },
  ]}
/>;
```

## Routing

### Subdomains

```
v2.docs.example.com → version 2
v1.docs.example.com → version 1
docs.example.com    → latest
```

### Subpaths

```
docs.example.com/v2/ → version 2
docs.example.com/v1/ → version 1
docs.example.com/    → latest
```

## SEO

### Canonical URLs

Always set canonical to the latest version:

```tsx
export const metadata = {
  alternates: {
    canonical: "https://docs.example.com/guide",
  },
};
```

### Sitemap priorities

```xml
<url>
  <loc>https://docs.example.com/guide</loc>
  <priority>1.0</priority>
</url>
<url>
  <loc>https://v2.docs.example.com/guide</loc>
  <priority>0.5</priority>
</url>
```

## Deprecation notices

Show banners on old versions:

```tsx
import { Banner } from "fromsrc/client";

{
  version !== "latest" && (
    <Banner variant="warning">
      you're viewing docs for {version}.<a href="/">switch to latest</a>
    </Banner>
  );
}
```
