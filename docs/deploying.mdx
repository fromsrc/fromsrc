Deploy fromsrc documentation to any platform.

## Vercel

Zero-config deployment:

```bash
vercel
```

Or connect your GitHub repo at vercel.com/new.

### Environment variables

Set in Vercel dashboard or `.env`:

```env
NEXT_PUBLIC_SITE_URL=https://docs.example.com
```

## Netlify

```bash
netlify deploy --prod
```

### Netlify.toml

```toml
[build]
  command = "bun run build"
  publish = ".next"

[[plugins]]
  package = "@netlify/plugin-nextjs"
```

## Cloudflare Pages

```bash
npx wrangler pages deploy .next
```

### Wrangler.toml

```toml
name = "docs"
compatibility_date = "2026-01-01"
```

## Static export

Generate static HTML for any host:

```ts
// next.config.ts
export default {
  output: "export",
};
```

```bash
bun run build
```

Deploy the `out/` folder to any static host:

- GitHub Pages
- S3 + CloudFront
- nginx
- Apache

## Docker

### Dockerfile

```dockerfile
FROM oven/bun:1 AS builder
WORKDIR /app
COPY . .
RUN bun install
RUN bun run build

FROM oven/bun:1
WORKDIR /app
COPY --from=builder /app/.next .next
COPY --from=builder /app/public public
COPY --from=builder /app/node_modules node_modules
COPY --from=builder /app/package.json .
EXPOSE 3000
CMD ["bun", "run", "start"]
```

### Docker Compose

```yaml
services:
  docs:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
```

## Self-hosting

### Standalone build

```ts
// next.config.ts
export default {
  output: "standalone",
};
```

Outputs minimal files needed to run:

```bash
node .next/standalone/server.js
```

### PM2

```bash
pm2 start .next/standalone/server.js --name docs
```

## Custom domain

### DNS configuration

```
docs.example.com  CNAME  your-host.com
```

### SSL

Most platforms handle SSL automatically. For self-hosted:

```bash
certbot --nginx -d docs.example.com
```
